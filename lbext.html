LioranBoard Extensions

Follower Alert

Creator: Lioran
Download: Preinstalled. Located in lioranboard/Extensions folder
Description: This extension is ready to be used as soon as you download LioranBoard, as it is preinstalled in your Transmitter. Twitch does not natively send follower alerts, so this is why there needs to be a separate extension to handle them.

(Skip these steps below if you installed Notification System as it is already included in it)

How to initiate the extension:

  1. Create a new button called Init Follower Alert
  2. Add a new command Send to Extension - Follower Alert:
    • Do not modifychannel_idvalue, unless you're using alternate accounts in LB. In that case you need to change it to your broadcaster's channel ID (which can be easily found by using this Chrome extension)
  3. Click on Done. Right click on the newly created button and select Add Twitch Triggers. Click on the + - Extension Trigger. Set it to !transmitteralways.
    This button will now get executed every time you refresh or close/reopen Transmitter, and will start listening for new followers.

How to set up a Follower trigger:

  1. Create a new button called Follower Alert.
  2. Add a new command Math: Trigger Pull . Set Variable to follower_name and Pull value to 1.
  3. Right click on the button and select Add Twitch Triggers. Click on the + - Extension Trigger. Set it to Follower Alert. The button will now get triggered every time the extension detects a new follower.
    We also recommend to check Add to Request Queue, which acts as a button queue. If there is another Follower Alert trigger event while the button is still active (for example if you get two followers at the same time), it will place it in the request queue and reactivate the button as soon as it's ready again.

And you're done! Every time you get a new follower, their name will be stored in the follower_name variable. You are free to add other commands to you Follower Alert button. For example, you can use the Twitch: Chat Message command to thank your new followers for joining your stream.

Note: The follower Alert is not instantaneous, as it periodically fetches Twitch data every 15-20 seconds. If you have a Streamlabs account, you can install a Streamlabs Alerts extension instead which also includes Follower Alert and has almost no delay.


Notification System

Creator: Lioran
Download: Located in lioranboard/LB Notification system folder
Description: LioranBoard comes with a fully functional notification animation system for Twitch, ready to use for OBS.
Preview:

Setup Instructions

  1. Copy the whole LB Notification System folder to LioranBoard Receiver(PC)\Extfolder.
  2. LioranBoard must be connected to OBS with 4.9 OBS Websocket installed.
  3. Install the "LioranBoard Notification System.lbe" extension found in this directoy to your Transmitter.
  4. There should be a new deck in your Receiver and a new scene in your OBS (which you can add as a source to your existing scenes). The notification system is not ready to be used. You can use the test buttons in your Receiver for various Twitch alerts.

The notification system comes with a TTS system. The default TTS will only work if the Transmitter is running in Chrome.

How to switch to Polly TTS (and be able to use it in your OBS dock):

  1. Install the Polly TTS extension.
  2. Go in the command list for "Master Animator" and "TTS voices" buttons and delete every command in there.
  3. Go in the "Alternate Master Animator" button, click the check box at the top left to select every command and click "Copy selected commands" and paste them in the normal Master Animator button.
  4. Repeat step 4 for Alternate TTS voices.
  5. The Polly TTS should now be fully integrated with your Notification system.

Note: If you have the StreamLabs Alerts extension installed, the notification system will pick up the donations by default.

Midi Keyboard

Creator: Lioran
Download: Discord Link
Description: Sends an extension trigger with every midi keyboard keypress. Transmitter needs to be running outside of OBS in Google Chrome.

Extension Triggers to expect:

  • midi keypress 0-127 - Any notes you press. Number will be the number of the notes. Math: Trigger Pull value 1=velocity.
  • midi key release 0-127 - Any notes you release. Math: Trigger Pull value 1=velocity.
  • midi volume - Math: Trigger Pull value 1= volume between 0-127
  • midi pitch- Math: Trigger Pull value 1= pitch bend 0-127

Timer

Creator: Christinna
Download: Discord Link
Description: You can use this extension to add as many timers as you want. Useful for triggering buttons (like sending chat messages) every X minutes. It is not precise to the second since it still needs to go through the websocket, so account for a variance of 5-10 seconds.

You can create as many timers as you want. The timers will clear only if you use Timer Stop extension, refresh Transmitter or close/reopen LB receiver.
You can keep editing your decks and buttons and even reset values, as this has no effect on the timers whatsoever.

How to set up a timer:

  1. Create a new button and add a command Send to extension - Timer.
  2. Enter the name of your timer. This will be the extension trigger that will fire each time your timer hits 0.
  3. Enter the delay in milliseconds. If you want to trigger a button every 5 minutes, it will be 60000 * 5 = 300000.
  4. Create another button or use an existing one you wish to trigger once the timer hits 0. Add a new extension trigger to the button which will be the name of your timer you set up in your previous step.
  5. Click done twice and initiate your timer by pressing the first button you created (press it only once, otherwise it will create duplicate timers).

Note: You can see all your currently running timers in your Transmitter by clicking on the "View Active Timers" button.


How to delete a timer:

  1. Create a new button and add a new command Send to extension - Timer Delete.
  2. The name will be the name of your timer you set up earlier (needs to match). StopAll can be either set to false (to only delete the specified timer) or true (to delete all currently running timers).

Note: This will delete your timer and start the countdown all over once you initiate the timer button again.


Auto Create Buttons

Creator: Christinna
Download: Discord Link
Description: This extension automatically creates a new deck and populates it with buttons that have predefined values so you don't have to manually create them one by one. The result can be something like this


List of all values you can configure:

  • deck file name - be careful to NOT OVERRIDE an already existing deck. Check your local folder and choose a deck file name that does not exist yet (each deck is formatted like this: configs/NUMBER/.ini). Do not skip any numbers or LB will not load it up. For example, if your last deck file is configs6.ini, LB expects the next one to be configs7.ini.
  • deck background color - use this color picker (GML Raw value).
  • grid height and weight - how many buttons you want each row and column to contain
  • default button color - use this color picker (GML Raw value). Applies to all the buttons.
  • default button text - applies to all the buttons
  • default button image - your image file name (do not include path). Needs to be located in Lioranboard/Receiver/image folder. Accepts .png files only. Applies to all the buttons.
  • button ID to start with - choose non existing button ID.

After you fill out all the fields, press the button and let it do all the magic. It might take some time if you are creating a lot of buttons. Once it is done, you can restart LioranBoard and the new deck should appear in your deck menu.
If you do not see your new deck, you probably did not select the right name for your configs/NUMBER/.ini deck file. Go back to your local folder, look up the highest configs/NUMBER/.ini file (for example configs15.ini) and rename your newly created deck file in the same fashion (for example configs16.ini). Reopen LB and it should be there.

This extension is just one premade button with commands, it does not matter which Transmitter you choose to install it in.

Censor Bad Words

Creator: Melonax & Christinna
Download: Discord Link
Description: This extension allows you to create a list of bad words you wish to get censored. It can be imported from .ini file by using File: Load String and Stack: Parse

INIT button

  1. Create a stack of bad words (case insensitive) you wish to be censored.
  2. Use Send to Extension command - Censor Bad Words:
    • type - INIT
    • stackname - name of your stack you just created
    • rest of the fields - leave empty
  3. Add !transmitteralways trigger to your button. This means it will be executed every time you refresh your Transmitter.

Your stack with the bad words gets cleared every time you use the INIT button to avoid any duplicate values.

How to censor a string

Once you create your Init button, you can use Send to Extension command - Censor Bad Words:

  • type - Censor
  • stackname - leave empty
  • string - string you wish to check for bad words
  • ReplaceBy - what to replace the bad word with
  • variable - name of the variable to save the new censored string


Get Folder Files

Creator: Christinna
Download: Itch.io
Description: This extension allows you scan a folder to retrieve all the file names and save them into a stack and .ini file. Very useful if you need to iterate through a lot files with different names and file types.
Setup: Place the whole Get_Folder_Files folder into your LioranBoard Receiver(PC)\Ext folder. There is a get_files.bat file that will be executed from LioranBoard. You can (and should) right click on it-edit to see the cmd commands: cd /d %1 navigates to your selected folder, dir /b /a-d outputs all the file names in the folder and |clip saves the output in your clipboard. Please make sure your antivirus did not delete or modify the batch file.

Follow the premade button instructions and fill out the following values:

  1. folder_path - full path to the folder which you wish to retrieve your file names from. "E:\Stream" or "E:/Stream" both work, but will be converted and saved as "E:/Stream".
  2. ini_file - name of the .ini file you wish to save your file names into. Will be created in lioranboard\LioranBoard Receiver(PC) folder if it doesn't exist.
  3. filter_types (optional) - a string of file types you wish to include, all other types will be ignored. Separated by comma. Example: "png,jpg,bmp"
  4. sort_order - sort your stack either alphabetically or numerically
  5. save_as - save the values either as file names (hello.png) or including the full path (E:/Stream/hello.png).

Do not edit any other values.
Once you press the button, it will take some time to execute, depending on the amount of files. If there are more than 500 files in your folder, LioranBoard might temporarily become unresponsive for up to several minutes. Just wait and let it do its own thing. Please note that there is a character limit, which corresponds to around 2000-3000 file names. It will not save all the files if it reaches the limit.

Important: Please do not use your clipboard to copy anything (CTRL+C) for at least 5 seconds just after you press the button. The button relies on clipboard functionality and it might not execute properly otherwise.

Once the button is executed, all file names are saved in the specified .ini file, including their stringified stack and amount. Please note the paths in the .ini file will always contain forward slashes, even if you use backward slashes in your file_path value.


You can use Math:Random and File:Load String commands to load a random file name, or you can use File:Load String and Stack:Parse commands to load up the whole stack of file names. Make sure to use forward slashes in your section name.


If the contents of your scanned folder change, you can freely rescan the folder, which will automatically override the file names in your .ini file (and delete all non existing ones).

If there are any errors, you should get a yellow notification message in your Receiver.

OBS Resizer

Creator: Christinna
Download: Itch.io
Description: OBS Resizer allows you to effortlessly resize your scenes, nested scenes, their sources and move source filters with one push of a button.
Preview:

This is an extremely useful extension if you need to switch to a different video resolution or you often create and share your scene collections with other people who need them for a different resolution. 

If you've ever had to change all your pixel-size perfect scenes to another canvas size, you know how much of a hassle it was and how long it took you to get it all back to perfection. Not anymore, one button does it all! 

As a bonus, the extension should automatically resize your crop and move source filters, so you don't need to worry about manually doing that either. 

It comes with a premade deck where you need to change only two values - your scene name and the original width you're resizing it from. After that you can just press the button and let all the magic happen. 

Important note:  Although the extension has been thoroughly tested, bugs sometimes happen! I strongly recommend duplicating your scene collection as a backup before using the extension. 

Setup:
After installing the extension, go to the premade deck and edit the Resize button:

  • SceneResize - name of the scene you wish to resize. All sources, nested scenes and their sources will be resized. Crop and Move Source filters should be also correctly resized.
  • OrigWidth - original width of the scene. For example, if you just switch from 720p to 1080p, the orignal width will be 1280.
  • Your current canvas width you're resizing your scene to will be automatically pulled from OBS. You do not need to set this one up.
  • obsws_port - your OBS websocket port. Should be 4444 unless you changed it.
  • obsws_pw - your OBS websocket password, if you set up any. Leave empty if you have no password.

Once you edit the Resize button, you can now proceed to press it. Don't forget to back up your scene collection first!

The extension automatically saves the last width you resized the scene to. If the saved value is different from what you manually entered, you might get a popup warning message. You can choose to edit the width value to match the one in the popup message, or ignore it and press the button again to proceed with your manually entered width. This is just a safety net in case of any typos.

Puzzle Game

Creator: Christinna
Download: Itch.io
Description: This extension allows you to transform any OBS source into a puzzle game. Your viewers can use the chat to swap the individual pieces to solve it!
Preview:

Dependencies:
LioranBoard Receiver 1.44
OBSWS 4.9.1
OBS Shader Filter: 1.21
StreamFX: 0.10.0.0
Move Transition: 2.4.4
Older versions might or might not work.

Important Stuff to know:

Editable variables - if there is an editable variable or optional command, it's marked with 'EDITABLE' comment.

puzzle_running - you can use this variable to check if a puzzle is running or not. 1 = running, 0 = not running.

oversized sources - please make sure you're not using overly oversized pictures or videos for your puzzle game. Their original size should not be much larger than your OBS canvas. Why? The extension creates a mirror source for each puzzle piece and it might unnecessarily tax your GPU. Downsize the original file if needed.

If you're running Transmitter outside of OBS in a browser (for example in Chrome), make sure it's the active tab (i.e. you can actually see the Transmitter) when you press Create puzzle button. If Transmitter is not the active tab, it might throttle the script and take quite a bit longer to create the puzzle.

Setup:
After installing the extension, go to the premade deck and follow these steps:

  1. set variables

    Edit 'set variables button':

    • obsws_password - your OBS websocket password. Leave empty if you do not use a password.
    • obsws_port - your OBS Websocket Port. Default is 4444.
    • puzzle_scene - name for your NEW puzzle scene that will be created. Make sure it doesn't exist already. Please note there is a bug in OBS where if you delete a scene and try to recreate it without completely restarting OBS, it will tell you it already exists.
    • puzzle_source - name of the source you're using to create the puzzle from. Can be your camera, random video, picture, etc.
      Please make sure you're not using overly oversized pictures or videos - their original size should not be much larger than your canvas in OBS for performance reasons.
    • puzzle_source_scene - name of the scene where your source resides.
    • puzzle_rows - number of rows you want your puzzle to have. Maximum is 26, but 3-4 is most likely what you're aiming for.
    • puzzle_columns - number of columns you want your puzzle to have. Columns and rows affect the difficulty of puzzle and also its performance - older streaming machines might not be able to handle something crazy as 10x10 puzzle grid (which is 100 puzzle mirror sources your GPU has to animate!).
      • The following are OPTIONAL values and can be left at default. Edit them for the best customization.
    • puzzle_border - whether you want a border around every piece. 1 = border, 0 = no border.
    • puzzle_border_thickness - how thick the puzzle border should be in pixels. Depending on how much you're going to resize the puzzle scene, you might want to increase the border thickness. If you set it to 1 pixel, but resize your puzzle scene to 1/3 of its original size, the border will partially or completely disappear. Play around with it and see what works for your case the best.
    • puzzle_border_margin - this affects the space between the pieces if border is present. Again, different settings might be needed for different resolutions and grid size.
    • puzzle_border_color - the color of the border, it accepts only color hex values.
    • puzzle_text_size - font size for the puzzle coordinates. This again depends on your much you resize your puzzle scene - increase/decrease the font size depending on your own settings.
    • puzzle_text_font - font name for the puzzle coordinates
    • puzzle_text_color - text color for the puzzle coordionates, it accepts only color hex values.
    • puzzle_text_outline - whether you want the text to have an outline. 1 = outline, 0 = no outline.
    • puzzle_text_outline_thick - thickness of the outline in pixels.
    • puzzle_text_outline_color - the color of the outline, it accepts only color hex values.

  2. create puzzle

    Once you're done editing the set variables button, press Done and again Done (twice total) to make sure it's saved.
    1. You must be connected to Transmitter, OBS and be on the scene with the source you want to create your puzzle game from. Make sure the source isn't hidden.
    2. If your Transmitter is running outside of OBS in a browser, make sure it's the active tab, otherwise the browser throttles its execution and it might take a while to create your puzzle.
    Press 'create puzzle' button.

  3. The create puzzle button should change to 'in progress'. Wait until the button changes to SUCCESS!, which means it finished creating the puzzle scene in your OBS (it might take longer depending on the size of the grid).
    You will get a popup message if there are any errors creating the puzzle.
    You will not see anything once the puzzle is created, because the whole puzzle scene has a Color Correction filter with its opacity set to 0. You can right click on the puzzle scene and disable the filter to check if it was created correctly.

  4. start game

    Edit the start game button variables:

    • puzzle_scene - the same name you chose for your new puzzle scene earlier
    • puzzle_sounds - whether you want to enable sound effects
    • puzzle_shuffle_mid - whether you want all pieces to go to mid when shuffling the puzzle.
    You can optionally delete the Statement: Exit if puzzle_running == 1 command, however I recommend leaving it in, so you don't accidentally start a puzzle game when one is already running already. Press Done and Done again (twice total).

  5. Press the start game button. The puzzle should automatically appear in your original scene and shuffle.
  6. Let your viewers play the game

    • move - use !puzzle move chat command to swap pieces, for example !puzzle A1 to C3.

    • shuffle - use !puzzle shuffle chat command to reshuffle the pieces. By default only broadcaster and their mods can use this command. Leaderboard will not be cleared.

    • solve - use !puzzle solve command to automatically solve the puzzle. By default only broadcaster and their mods can use this command. Leaderboard, variables and stacks will be all cleared.

    • leaderboard - use !puzzle leaderboard to post the name of the viewers with the most correct moves. The default number of viewers shown is 5. You can change this value inside the button.

    • reset - use !puzzle reset to completely clear the leaderboard and all variables and stacks. This button is automatically pressed when puzzle is started or solved.
