Basic Setup

Prerequisites

  • OBS Studio Download link
    LioranBoard is to be used in combination with OBS Studio. It does not work with Streamlabs OBS.
  • OBS Websocket Download link
    This is how LioranBoard communicates with OBS. We recommend installing the latest version to avoid any issues.

Installation

Complete installation instructions:


After downloading and extracting the .zip file, you will see a few different folders:

  • LioranBoard Receiver
    This folder contains the main editor called LioranBoard Receiver.exe, where you set up all the important connections, create buttons and commands. This needs to be running at all times for LioranBoard to work.
  • LioranBoard Stream Deck (PC)
    This is your PC Stream Deck which connects to the Receiver. From here you can trigger your buttons that you created in the Receiver.
  • LioranBoard Stream deck (Android)
    This is your Android Stream Deck which can be used to connect to Receiver on any Android device.

For now you can ignore the other folders.
It is recommended to not install LioranBoard in Program Files folder as it sometimes tends to block LioranBoard from writing in its own folder.

Transmitter

Transmitter (tsl_transmitter.html) is a simple .html file which allows LioranBoard to communicate with external sources, including Twitch and all the extensions. It needs to be running at ALL times and always only ONE instance, as LioranBoard does not allow more than one connection.

The Transmitter is divided into tabs, each representing one extension. You can freely organize them and also completely hide them (by ticking/unticking the checkboxes). There are different themes you can choose from as well. The main tab Pubsub Alerts contains buttons and input boxes that you can use to test various Twitch alerts (once you set them up in your Receiver), coming from either Twitch Pubsub or Twitch Chat. Please note that Follower Alerts need to be tested in a separate extension.

IMPORTANT: Transmitter must be placed outside of LioranBoard Receiver folder to function properly. Do not put it in the same folder as Lioranboard Receiver.exe, or a folder within that folder.

Adding Transmitter to OBS

There are three different ways to use Transmitter:

  1. As a dock in OBS
    This is the simplest and most effective way. You can create a dock by clicking on View-Docks-Custom Browser Docks and fill in the path to the file. Click on apply and place the dock wherever it suits you in OBS. You will need to keep it open at all times.
  2. As a browser source in OBS.
    This is highly discouraged as it is easy to lose track of it and accidentally get two transmitters running at the same time (which you then have to locate and disable/remove, else LB will not work).
  3. Running it in your browser outside of OBS.
    This is needed in some cases, for example if you want to use Text to Speech extension.

If you did everything right, there should be a green text saying “Connected to LioranBoard” inside your Transmitter.
You should also see a yellow notification message in your Receiver saying “TSL Transmittor is connected”.
If you see an error message that second Transmitter tried to connect, you need to locate and close it for LB to work properly. If your Receiver crashes when you try to connect to the Transmitter, make sure you are on the latest version. Some improperly coded extensions can also crash your Receiver.

Connecting to OBS websocket

If you installed all the necessary dependencies, go to your OBS-Tools-Websockets Server Settings and make sure Enable Websockets server is checked. You can also enable authentication and set up a password if desired.

Launch your LioranBoard Receiver and click on Connect to OBS. If you set up a password in the earlier step, you also need to click on Options and fill out OBSwebsocket Password field.

If connected successfully, the Connect to OBS button will turn gray and you will see a yellow notification message at the bottom of your Receiver, shortly followed by another yellow message "OBS data have been fetched".

Linking your Twitch account

Open LioranBoard Receiver and click on Link your Twitch. You can select which Authorization scopes (permissions) you want LB to be able to use. Default values are recommended, unless you're installing a new extension that requires some additional ones. Next step you need to do is click on Open URL.

Click on Authorize and LB should do the rest.

You can then click on Connect to Twitch in your Receiver to test your connection (Transmitter must be running).
You should see these following yellow notification messages and it is important to check that ALL of these messages actually come up:

  • Twitch is connected
  • Listening for Channel points
  • Listening for Bits donations
  • Listening for Channel subscriptions
  • Listening for Chat Moderation
  • Connected to Twitch chat.

Congratulations, your Twitch account is now connected.

Renewing your Twitch token

You might get a pop message saying that your Twitch token is expired.

This is a safety feature implemented by Twitch - your token expires every 2 months.
In this case you want to click on Link your Twitch, press the Unlink button and follow the same procedure for linking your Twitch account.

Connecting to Stream Deck on PC

This is as simple as going into your LioranBoard Stream Deck(PC) folder, launching LioranBoard Stream Deck and clicking on Connect. You should not need to change any of the default settings.

Connecting to Stream Deck on Android devices

LioranBoard Stream Deck only works with Android devices (Android 2.3 Gingerbread and above) and is not available in Google Store. You need to manually transfer the .apk file found in your LioranBoard Stream deck(Android) folder to your Android device and install it (you might need to change permissions to install external files).
On your computer, go to LioranBoard folder and run whats my local ip.bat file. Write down the IPv4 Address (it should start with 192.168.)
Use it as an IP address for your Android Stream Deck connection. Port should stay 9423, unless you changed it in your Receiver.
If you're having trouble connecting with your Android Stream Deck, please consult the troubleshooting section and also this Google Doc guide created by Leggolem.


Deck and buttons

You can either use one of the premade decks or create a new one by clicking on Add New Deck. Remember that decks are just there to make your buttons more organized. As long as the Receiver is running, every single deck and its buttons are active. After creating a new deck, you can change its name, background color and grid size.

Adding a new button
Right click anywhere on the grid and select Create Blank Button or choose one of the popular commands. Name your button and click Create. You can freely resize it, change its color, picture and text as well.


There are a few options that pop up by right clicking on the newly created button:

  • Change color/text/image
    Pretty self explanatory. One thing to note is that all images you add are immediately copied over to Receiver/image folder and resized. Png format only.
  • Add keyboard macro
    This adds a keyboard macro to your button that will be executed when the button is pressed. Only keyboard keys are supported, including f13 to f24 keys. This is useful for triggering actions in your other programs (both Receiver and the other program needs to be running in admin mode for it to work). Keyboard macros have nothing to do with your button commands.
  • Add sound effects, sound mods, stop all sounds
    Receiver only supports .ogg files. You can use Audacity to convert your sound files to .ogg files. LioranBoard's sound system is quite limited and it's often more practical to trigger sound files through OBS (Source Change Settings command) rather than LioranBoard itself.
  • Add/Edit Commands
    This is where the magic happens. This is your empty canvas where you create your own code which gets executed when you press the button. You can use parameters, conditions, delays, pull data from OBS websocket and Twitch, do some pretty complex math operations etc.
    • In order to save your new commands, make sure to always click Done twice (once in your button editor and once in your deck editor screen).
    • You can use CTRL+Z to undo an action, it remembers the last 5 actions (some quick actions such as pasting commands in rapid succession count as a single action). Undo action is available in version 1.43 and newer.
  • Add Switch Deck
    This allows you to easily switch between decks in your Stream Deck.
  • Add Twitch Triggers
    You can specify conditions that trigger your button. For example, you can make a button that will trigger with a Twitch chat message !command or whenever someone raids you. Wildcards can be added as well.
  • Add Drag press
    None: Activate when pressed
    Drag: Activate when pressed or moved over
    Multiple Drag: Same as Drag, but it will activate every time it's moved over again without lifting the finger.
  • Copy/Delete button
    All buttons have unique IDs. If you copy a button, it will get a new ID different from the original one.
  • Export/Import JSON
    You can use this feature to share your buttons with other people.
    Click on Export JSON, which will be saved to your clipboard. You can then directly send it to someone or use Pastebin.com.
    Once you have the JSON string ready in your clipboard, you can then use Import JSON and the button will be copied over.

  • Button ID
    This is a unique identifier for each button. If you copy and paste a button, it will always have a different ID (unless you deleted the previous one). Button ID is required if you want to set up Trigger Shortcut for the button or use some specific commands, like Trigger Button, Modify Button or Clear Button. You can change the ID of a button by clicking on it. It accepts a maximum of 10 characters and can contain numbers, letters and special characters.

Menu Options

The Options menu in the main LioranBoard Receiver screen can be mostly left at default values.

  • Listen Port - a port Receiver listens to for incoming traffic from StreamDeck and Transmitter. Do not change unless you need to use multiple instances of the Receiver.
  • OBSWebsocket Port - a port for OBS Websocket connection. Do not change, unless you also changed it in your BS-Tools-OBS Websocket Server Settings.
  • OBSWebsocket Password - a password for OBS Websocket. Leave blank, unless you set one up in your OBS-Tools-OBS Websocket Server Settings.
  • Auto connect to OBSws - whether you want the Receiver to automatically connect to your OBS websocket as soon as you start it up.
  • Auto connect to Twitch - whether you want the Receiver to automatically connect to Twitch as soon as you launch Transmitter.
  • Non blocking connection - change only if you're trying to troubleshoot your OBSWS connection. It should be disabled on default.
  • Frame Per Second - Framerate at which your Receiver runs. This greatly affects all the animations it performs, including the ones in OBS. Maximum is 60 FPS.
  • Reset Values and running buttons - completely resets all your variables and clears all the currently running buttons

Extensions

Extensions are meant to extend or simplify what LioranBoard is able to do. They're mostly written in JS and installed in your Transmitter. You can find a list of extensions here, although our Discord server is the best source for all up-to-date extensions.

Install an extension:

  1. Download the .lbe extension file
  2. Click on Install Extension in your LioranBoard Receiver
  3. Select the extension file you downloaded
  4. Select your default Transmitter you are using. Make 100% sure it is the correct one.
  5. Your Transmitter will automatically refresh, adding a new tab for the extension you just installed. You should also see it added in at the bottom list of all extensions (with checkboxes).
  6. Most extensions include a premade deck with buttons. If you do not see one, create a new button, add "Send to Extensions" command and select the extension you just installed.

Note: Some extensions have built in requirements, such as placing them in the Receiver/Ext folder or running a compatible version of LB/OBS. If you do not meet the requirements, you will receive a popup message and the extension will not be installed to prevent incorrect use.

Extension is installed properly if:

  • You can see all its fields in your Send to extension command. If you can only see the extension name, the extension was not installed properly or your Transmitter is not running. The amount of input fields depends on the extension, but it should in most cases contain at least one.
  • Your Transmitter displays that it's correctly installed (you should see a new tab and a checkbox at the bottom with the extension name added).

Uninstall an extension (version 1.43 and later):

  1. Click on the 'Uninstall Extens.' button in your Receiver.
  2. Select the correct Transmitter where your extension is installed and press yes when prompted whether you want to replace it.
  3. You will get a dropdown menu in your Receiver. Click on the extension you wish to uninstall and press Yes.
  4. Your extension is now uninstalled and you can no longer use it.

Notification system

LioranBoard comes with a fully functional notification animation system for Twitch, ready to use for OBS. See here what it looks like:


Setup Instructions

  1. Copy the whole LB Notification System folder to LioranBoard Receiver(PC)\Extfolder.
  2. LioranBoard must be connected to OBS with 4.9 OBS Websocket installed.
  3. Install the "LioranBoard Notification System.lbe" extension found in this directoy to your Transmitter.
  4. There should be a new deck in your Receiver and a new scene in your OBS (which you can add as a source to your existing scenes). The notification system is not ready to be used. You can use the test buttons in your Receiver for various Twitch alerts.

The notification system comes with a TTS system. The default TTS will only work if the Transmitter is running in Chrome.

How to switch to Polly TTS (and be able to use it in your OBS dock):

  1. Install the Polly TTS extension.
  2. Go in the command list for "Master Animator" and "TTS voices" buttons and delete every command in there.
  3. Go in the "Alternate Master Animator" button, click the check box at the top left to select every command and click "Copy selected commands" and paste them in the normal Master Animator button.
  4. Repeat step 4 for Alternate TTS voices.
  5. The Polly TTS should now be fully integrated with your Notification system.

Note: If you have the StreamLabs Alerts extension installed, the notification system will pick up the donations by default.

Update and Data Backup

Update to a new version without losing any data
Replace LioranBoard Receiver.exe and data.win with the new files.
To update the Stream Deck, you can just replace the files.
In some cases you will also need to replace your Transmitter file (if there is an update) and reinstall all your extensions. Replacing your Transmitter will not modify any of your decks or buttons.

How to back up data
There is a small text at the bottom in LioranBoard Receiver which says either local or Appdata.

  • Local - go to Lioranboard Receiver folder and copy all the configs.ini and connect.ini files. You can also back up images and sounds folder.
  • Appdata - go to C:/user/%username/appdata/local/lioranboard and copy all the files.

Alternate Twitch account

There has been a change in the new LioranBoard version. You will NOT automatically connect to Twitch Chat anymore with your secondary LB accounts (your 'main account' is the name of your board). Instead, you need to use the new command Twitch: Join Channel. We recommend putting !transmitteralways trigger on the button with this command and adding 10000 ms delay to give the Transmitter time to connect to chat. This will automatically trigger it every time you launch Transmitter and connect your alternate accounts to chat. Everything else works the same.

You can link an alternate Twitch account to your Receiver (and effectively make it the main LB account instead) if you don't want LioranBoard to be using your stream account to send Twitch chat messages. It makes it easier for your viewers to tell the difference between you personally interacting with them and any automated messages you have set up in LB.

  1. In your Receiver, click on Link your Twitch - Unlink and close the menu.
  2. Make sure you are logged in with your alternate/bot Twitch account in your browser at https://twitch.tv. Link your Twitch bot/alternate account first by following the same steps in Linking your Twitch account section.
  3. Once your bot account is linked successfully, go back to your browser, sign out and log in with your main Twitch account.
  4. In LB Receiver click on Link your Twitch again, but this time do NOT open the URL directly. Instead Copy it and close the window (ignore the warning and press Yes).
  5. Go back to your browser (make sure you're logged in with your main Twitch account) and paste the URL you copied earlier. Once you accept the permissions, you will be redirected to an empty page with an URL that looks like this: http://localhost:29521/tokenhtml#access_token=XXXXXXXXXXXXXXXXXXXXXXXXXXXXXX&scope=bits... Inside the URL is your access token (excluding the & sign). Copy it.
  6. Open connect.ini file in your Lioranbord Receiver or Appdata/Local/Lioranboard folder. You want to add these lines to your file:
    [account] 0="account_name" 0_oauth="XXXXXXXXXXXXXXXXXXXXXXXXXXXXXX"
  7. Save the file and restart LioranBoard Receiver. If you did everything right, you should see four yellow messages that say: Listening for alternate account... after LB connects to your Transmitter.

When using the command Twitch: Chat Message, you will now need to specify the channel name, else it will be sent to your main LB account's (which is the first one you linked and the name of your board now) channel. You will need to renew your tokens every 2 months. Keep in mind that LB doesn't offer a way to tell where the Twitch triggers come from, whether it's your bot or main account.
