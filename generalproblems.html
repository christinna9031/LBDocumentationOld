General

Crashes On Launch

Most likely Antivirus/Firewall problem. Try allowing the app through and/or deleting gmfile.dll in your LioranBoard Receiver folder.
If you are running Voicemeeter, you need to start it AFTER LioranBoard. Alternatively, the default output audio device needs to be set to the Voicemeeter virtual device.
Sound drivers can be also causing this issue, you can try reinstalling them.

OBS Connection Issues

You can try these steps to troubleshoot your OBS Websocket connection to LB:

  1. Redownload OBSwebsocket
    If in doubt, try completely reinstalling the plugin. Make sure you have the latest version. You can check whether your OBS websocket is correctly installed by going to OBS-Tools-Websockets Server Settings
  2. Check OBS websocket password
    Make sure your passwords in OBS-Tools-Websockets Server Settings and LioranBoard Receiver-Options match. If you do not use a password, uncheck Enable authentication.

    >
  3. Change Ports
    Change the ports of OBSwebsocket to a new one, try a random number like 19838.
    Make sure you also change the port for OBSwebsocket in LioranBoard to the SAME number, the box labeled OBSWebsocket Port inside Options.
  4. Lower or increase timeout
    Open connect.ini (either in Lioranboard Receiver or in APPDATA/local/lioranboard folder).
    Add timeout="1000" under [connection] . Try 100 and 5000. At 100ms you can just spam the connect to OBS button see if that helps..
  5. Try Non Blocking Connection
    In your Receiver, go to options and check/uncheck non blocking connection.
  6. Change IP connection
    Normally you use 127.0.0.1 to connect to OBS, but you can try local IP as well.
    Run whats my local ip.bat located in Lioranboard folder and copy your IPv4 address (it will start with 192.168). Open connect.ini file and add websocket_ip="192.168.2.10" under [connection]
  7. Running as Admin or non-Admin
    You can test both scenarios to see if it helps.
  8. Make sure your firewall is not blocking LB
    You can try completely disabling your antivirus/firewall or adding an exception to allow the app through.
  9. Try connecting using other means
    You can try this Chrome Extension.

Twitch Alerts don't get triggered at all
First, make sure your Transmitter is connected to LioranBoard. If so, try triggering a test alert from your Transmitter.

A small yellow notification message should pop up at the bottom of your Receiver:

  • If you do not see the message at all, your Transmitter is not connected to the Receiver. You might be running a wrong version or duplicate instances of your Transmitter (only a single instance can be running at the same time).
  • If you see the message, but your buttons do not visually trigger at all, your button triggers are set up wrong. Please follow this guide.
  • If you see the message and your buttons visually trigger, but your commands do not trigger anything in OBS or do not send a Twitch message, you're not connected to OBS/Twitch. Read OBS connection troubleshooting or Twitch connection troubleshooting section.
  • If you see the message, your buttons visually trigger and your commands are executed right, you're not connected to Twitch. Try relinking your Twitch account and restarting Transmitter. You should see these following yellow messages at the bottom of your Receiver:
    • Twitch is connected
    • Listening for Channel points
    • Listening for Bits donations
    • Listening for Channel subscriptions
    • Listening for Chat Moderation

Twitch alerts work, but Twitch chat does not send or trigger any messages
Twitch Alerts and Twitch Chat are two different separate connections. Twitch Alerts are managed by PubSub and Twitch Chat is managed by Twitch IRC.
Restart your transmitter and make sure you see this yellow message: Connected to Twitch chat.
If you do not see it, try relinking your Twitch account.

Transmitter Connection

Error message "A second Transmitter attempted to connect. Please close it."
You might encounter this error message in your LB Receiver.
It means that you have two or more Transmitters open at the same time, which is a big no-no in LioranBoard world.
You need to locate the duplicate Transmitter and close it in order for LioranBoard to work correctly. The most likely culprit is another Transmitter added as an OBS source and forgotten about.

Deck Creation Issues

LB crashes when I try to create a new deck.
Probably an antivirus issue. Try deleting gmfile.dll in your LioranBoard Receiver(PC) folder. Could be also a folder permissions problem. More info below.

Newly created decks and/or buttons are not saving.
This is an antivirus problem. LioranBoard's default file location is in lioranboard/Receiverfolder. If something blocks it from writing in this folder, it will attempt to save it in C:/user/%username/appdata/local/lioranboard folder instead.

  1. Try disabling your Antivirus and/or Firewall. You can also try changing folder permissions.
  2. Check these two locations: lioranboard/Receiver and C:/user/%username/appdata/local/lioranboard folder.
    Only one of these locations should contain configs.ini files. If both folders contain them, back them up and delete the files from both locatiions. Restart LioranBoard and see if this step helped.

Unresponsive Buttons

If none of your buttons work, you are probably not actually triggering them.
There is a common misconception where newcomers try to trigger their buttons from inside LB Receiver instead of Stream Deck.
Your Receiver only acts as an editor. If you want to press a button, you need to open Stream Deck and trigger it from there instead.

Android Stream Deck Troubleshooting

Android Stream Deck will not connect.
If your Stream Deck crashes or you can only see a green screen without any buttons when you try to connect to it, try these steps:

  1. Make sure your Android device is connected to the same network as your computer that is running LB Receiver. You cannot natively make a connection outside your network, unless you decide to go through a lot of technical steps involving opening your ports and exposing your local network.
  2. Make sure you are using the correct IP. Run whats my local ip.bat in your lioranboard folder and double check your IPv4 Address. It will start with 192.168. Anything starting with 127.0. is a localhost IP address which only works on the same computer that runs your LB Receiver.
    Auto correct on Android devices can also be the culprit.
  3. Try completely disabling your Antivirus/Firewall and pinging your IPv4 address from your Android device to see if you get any response back. You can use a PingTools app for that.
  4. If none of this helps, you can try changing ports and going to your router settings to see if anything could be blocking the connection.

You can also consult this Google Doc guide created by Leggolem.

Keyboard Macros

Keyboard macros are not working. OBS (or any other app) completely ignores them.
Try running both your Receiver and the other app as Administrator. Run your Stream Deck as non administrator.
If you are still having problems, try going to a keyboard tester website, like this one. In your Receiver create a new keyboard shortcut button with 2000ms delay. As soon as you trigger the button, switch to the Keyboard tester page to see if it actually pushes the button. If it does, the other app might be ignoring the input for some reason.

Installing Extensions

I installed an extension, but do not see it anywhere.
After installing any extension, your Transmitter should automatically reload (if you have the newest LB version) and you should see the installed extension there. Please follow How to install or uninstall extensions guide.
You can also check write permissions for the transmitter.html file.

Other Apps

There are some apps that like to interfere with LioranBoard.
Voicemeeter can crash LioranBoard or make its sound system not work. Make sure you always open LioranBoard first.
