OBS Source Commands

Scene Switch

Switch to the selected scene with normal transition.

Name Type Description
scene-name String Name of the scene in OBS

You can switch to a previous scene by using the premade variable previous_scene

Change Scene Transition

Temporarily changes the transition of a scene. Lasts until you use Remove Scene Transition command.

Name Type Description
sceneName String Scene name you are going to switch to
transitionName String Transition name to use
transitionDuration Int Duration of the transition in milliseconds

Remove Scene Transition

Removes a temporarily added transition.

Name Type Description
sceneName String Scene name that has a temporary transition added to it

Source Mute

Mutes the sound of a source.

Name Type Description
source String Source name to be muted
mute boolean True=mute or false=unmute

Source Toggle Mute

Toggles on and off the sound of a source

Name Type Description
source String Source name

Change Audio Monitor Type

Changes the monitoring type of an audio source. Same option as in OBS-Audio Mixer-Cogwheel-Advanced Audio Properties.

Name Type Description
sourceName String Audio source name
monitorType dropdown menu Type of monitoring to use

Source Change Visibility

Changes the visibility of a source.
It is highly recommended to use True or False. Toggle should be used only if it's a single button command.

This command does not always properly work if the source is in a group. It is recommended to use nested scenes instead of groups.

Name Type Description
scene-name String (optional) Scene name containing the source. Leave blank for current active scene.
item String Source name
visible boolean True, False or Toggle

Source Change Position

Changes the position of a source.
This happens immediately. If you prefer to have a smooth transition, use Source Motion: Position command instead.

Name Type Description
scene-name String (optional) Scene name containing the source. Leave blank for current active scene.
item String Source name
Field: X Real (optional) X position to set the source to. Can leave blank to not modify it.
Field: Y Real (optional) Y position to set the source to. Can leave blank to not modify it.

Source Change Volume

Changes the volume of a source.
If you prefer the volume to change gradually, use Source Motion: Volume command instead.
You can use Math: Pull Source Value command to get the current volume of a source and incorporate it in this command.

Name Type Description
source String (optional) Scene name containing the source. Leave blank for current active scene.
volume % Real Percentage to set the volume to, between 0 and 100.

Source Change Scale

Changes the size of a source.
You can use negative values for a horizontal (mirror) or vertical source flip.
The way it's resized is based on source's positional alignment in OBS (right click on the source in OBS-Transform-Edit Transform).
If you prefer the size to change gradually, use Source Motion: Scale command instead.

Name Type Description
scene-name String (optional) Scene name containing the source. Leave blank for current active scene.
item String Source name to change size.
X Scale Real (optional) X Scaling to set the source to. Can leave blank to not modify it. 1.0=100%
Y Scale Real (optional) Y Scaling to set the source at. Can leave blank to not modify it. 1.0=100%

Source Change Rotation

Changes the rotation of a source.
The center point of the rotation is based on the Positional Alignment option (right click on the source in OBS-Transform-Edit Transform).
If you prefer the rotation to change gradually, use Source Motion: Rotation command instead.

Name Type Description
scene-name String (optional) Scene name containing the source. Leave blank for current active scene.
item String Source name to change rotation.
rotation Real Rotation angle to change the source to. 0=normal, 180=turned around, 360=normal.

Source Change Settings

Changes settings of a source.
You need to know the exact formatting for this to work.
I recommend using a command Custom Packet first to find out what's the exact name of the value you're trying to change:
{"request-type":"GetSourceSettings","sourceName":"YOURSOURCENAME","message-id":"6666"}
Once you execute this particular command, you will get a grey message containing all source's settings. This will be automatically saved to your clipboard, so you can paste in a document for a later review. You can then find the exact sourceSettings value to use.


Name Type Description
sourceName String Source name to change settings.
sourceSettings JSON The settings to be included. This is a JSON format, {} must be present on both sides.

Most common sourceSettings values are:

  • Change URL of a browser source: {"url":"http://twitch.tv"}
  • Change local file path of a browser source: {"local_file":"E:/Stream/hello.html"}
  • Change local file path of an image source: {"file":"E:/Stream/image.png"}
  • Change local file paths for your VLC source. Each {"value":"X"} represents one file.
    {"playlist": [{"value": "E:/videos/video1.mp4"},{"value": "E:/videos/video2.mkv"}]}
  • Change font settings of your GDI+ text source. Needs to include all values.{"font": {"face": "Arial","flags": 8,"size": 48,"style": "Regular"}}

Pay special attention to the forward slashes, as it will not work if you use back slashes instead.

Text GDI+ Change Text

Changes the text of a Text GDI+ source.
If you wish to change the text to a value saved in a variable, you need to wrap it like this: /$variable$/

Name Type Description
sourceName String Text GDI+ source name
text String The new text

Freetype 2 Change Text

Changes the text of a Freetype 2 text source.
If you wish to change the text to a value saved in a variable, you need to wrap it like this: /$variable$/

Name Type Description
sourceName String Freetype 2 source name
text String The new text

Filter Change Visibility

Changes visibility of a filter in a given source.
It is highly recommended to use True and False. Toggle should be used only if it's a single button command.

Name Type Description
sourceName String Source name containing the filter
filterName String Filter name to change the visibility
filterEnabled boolean Enable or disable filter's visibility. On/off or toggle.

Filter Change Settings

Changes or adds settings to a filter in a specified source.
You need to know the exact formatting for this to work.
I recommend using a command Custom Packet first to find out what's the exact name of the filter settings you're trying to change:
{"request-type":"GetSourceFilterInfo","sourceName":"YOURSOURCENAME","filterName":"YOURFILTERNAME","message-id":"6666"}
Once you execute this particular command, you will get a grey message containing all filter's settings. This will be automatically saved to your clipboard, so you can paste in a document for a later review. You can then find the exact filterSettings value to use.


Name Type Description
sourceName String Source name containing the filter
filterName String Filter name to change the settings
filterSettings JSON The settings you want to set on the filter. JSON format.

Source Motion: Position

Moves a source from point A to point B over a given duration.
If you want to move a source from its current position that is not always the same, you can use the command Math: Pull Source Value to find the current source's X and Y position and incorporate it into this command.

Name Type Description
scene String Scene name containing the source
source String Source name to move
Start X Real Starting X position
Start Y Real Starting Y position
Final X Real Final X position
Final Y Real Final Y position
Dur.(MS) Int Duration of the transition in milliseconds
Smooth dropdown menu Type of transition. None= Normal, Out=Starts out fast and ends slow, In=Starts out slow and ends fast,
In/Out= Starts out slow, gets faster and then slows down at the end again.

Source Motion: Position Curve

Moves a Source from point A to point B over a given duration AND curves it in a circle pattrern.
You can make a source complete a half circle or have a slight curve in its transition.
If you want to move a source from its current position that is not always the same, you can use the command Math: Pull Source Value to find current source's X and Y position and incorporate it into this command.

Name Type Description
scene String Scene name containing the source
source String Source name to move
Start X Real Starting X position
Start Y Real Starting Y position.
Final X Real Final X position
Final Y Real Final Y position
Curve % Real The amplitude of the curve. Can be negative or over 100.
100% is a perfect half circle, 0% is straight line with a slower start and slower finish.
-200% is an elongated half circle on the opposite side.
Dur.(MS) Int Duration of the transition in milliseconds.

Source Motion: Scale

Resizes a source from Size A to size B over a given duration.
Due to the way the OBS source resizing work, the result might look a bit snappy sometimes.
The way the source is resized is based on what the source's point of origin is set to in OBS.
You can use negative values for a horizontal (mirror) or vertical source flip.
You can use Math: Pull Source Value command to find current source's X and Y size and incorporate it into this command.

Name Type Description
scene String Scene name containing the source
source String Source name to move
Start Xscale Real Starting X size
Start Yscale Real Starting Y size
Final Xscale Real Final X size
Final Yscale Real Final Y size
Dur.(MS) Int Duration of the transition in milliseconds
Smooth dropdown menu Type of transition. None= Normal, Out=Starts out fast and ends slow, In=Starts out slow and ends fast,
In/Out= Starts out slow, gets faster and then slows down at the end again.

Source Motion: Volume

Changes the volume of a given source from volume A to B over a duration.
You can use Math: Pull Source Value command to get the current volume of a source and incorporate it in this command.

Name Type Description
source String Source name to change the volume
Start Volume% Real Starting volume percentage. 0-100
Final Volume% Real Final volume percentage. 0-100
Dur.(MS) Int Transition duration in milliseconds

Source Motion: Alpha Filter

Changes the transparency of a source over a set duration.
Requires you to add a Color Correction filter to your source.
This way you can gradually fade out a source before turning its visibility completely off and gradually fade in a source after turning its visibility on.

Name Type Description
source String Source name to change the transparency of.
filterName String Filter name. Must be a Color Correction filter.
Start Alpha% Real Starting transparency percentage. 0-100
Final Alpha% Real Final transparency percentage. 0-100
Dur.(MS) Int Transition duration in milliseconds

Source Motion: Rotation

Changes the rotation of a source over a set duration.
The center point of the rotation is based on the Positional Alignment option (right click on the source in OBS-Transform-Edit Transform).
You can use Math: Pull Source Value command to get the current rotation of a source and incorporate it in this command.

Name Type Description

Final Rotation Real Final Rotation. 0-360
Dur.(MS) Int Transition duration in milliseconds

In/Out= Starts out slow, gets faster and then slows down at the end again.

Custom Packet

Sends a custom packet to the OBS websocket plugin in JSON format.
All the possible requests are documented in OBS websocket protocol reference.
Message-id 6666 means you will get a pop up message on your screen every time you send the packet (useful for troubleshooting). If you do not wish to see this, use message-id 1.

Name Type Description
String JSON JSON string to send to obswebsocket. Comes with an example to toggle studio mode.

A few examples of various requests:

  • {"request-type":"GetVersion","message-id":"6666"} - current OBS websocket version, status and available requests
  • {"request-type":"GetStats","message-id":"6666"} - OBS stats (similar info as provided in OBS's stats window)
  • {"request-type":"SetCurrentProfile","profile-name":"YOURPROFILENAME","message-id":"1"} - changes active profile in OBS
  • {"request-type":"StartRecording","message-id":"1"} - starts recording
  • {"request-type":"StartReplayBuffer","message-id":"1"} - starts replay buffer
  • {"request-type":"GetSceneList","message-id":"6666"} - retrieves a list of scenes in the current profile
  • {"request-type":"GetSceneItemProperties","scene-name":"YOURSCENENAME", "item":"YOURSOURCENAME","message-id":"6666"} - retrieves source item properties (like position, rotation, scale etc.)
  • {"request-type":"GetSourceFilterInfo", "sourceName":"YOURSOURCENAME", "filterName":"YOURFILTERNAME", "message-id":"6666"} - retrieves current source filter settings

Fetch OBS Data

Allows you to request data from OBS websocket and save it in a variable.
All the possible requests are documented in OBS websocket protocol reference.
Keep in mind that this is over a TCP connection, which means the value will not be saved immediately (give it 100ms delay).

If the requested value is inside another object already, you can access it with a simple dot-notation: ..
For example, if you want to retrieve Brightness value from Color Correction Filter, you will notice that it is inside an object called settings. {"settings": {"brightness": 0.78}, "status": "ok", "type": "color_filter"}. In this case, the Fetch Value will be settings.brightness.

Note: If the name of the fetched value contains dots, you need to wrap in a parentheses like this: (Filter.Transform.Rotation.X). Otherwise LioranBoard will think you're trying to access a value inside an object.

Name Type Description
OBS command JSON Works the same as the custom packet command, but you need to omit the "message-id" part.
Instead of sending {"request-type":"GetStats","message-id":"1"}, you will just send {"request-type":"GetStats"}.
Fetch Value String or object Name of a value you wish to get. If it's an object value, you need to include that.
Variable String Name of a variable to save the data under.
Turn to real boolean Whether you expect a real value (=number) or a string.

A few command examples:

  • Retrieve current width of a source
    OBS command: {"scene-name":"YOURSCENENAME","item":"YOURSOURCENAME","request-type":"GetSceneItemProperties"}
    Fetch value: width
  • Get CPU usage
    OBS command: {"request-type":"GetStats"}
    Fetch value: stats.cpu-usage
  • Get Text GDI+ source content
    OBS command: {"request-type":"GetTextGDIPlusProperties","source":"YOURSOURCENAME"}
    Fetch value: text
  • Time elapsed since streaming started
    OBS command: {"request-type":"GetStreamingStatus"}
    Fetch value: stream-timecode
  • Get a Render delay filter value
    OBS command: {"request-type":"GetSourceFilterInfo","sourceName":"YOURSOURCENAME","filterName":"Render Delay"}
    Fetch value: settings.delay_ms
  • Get an opacity value for Color Correction filter
    OBS command: {"request-type":"GetSourceFilterInfo","sourceName":"YOURSOURCENAME","filterName":"Color Correction"}
    Fetch value: settings.opacity
